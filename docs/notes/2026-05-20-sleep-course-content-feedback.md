## 🟡 建議釐清 / 補強

### 1. Day 8-14 進階機制請定義明確

「使用方式」說：「Day 8–14 依 Day 7 體感結果維持、降低難度或調整加速任務」

但沒明說：
- 使用者**如何評估 Day 7 體感**？
- 「降低難度」**具體怎麼操作**？

**建議方案**：Day 7 結束 app 推播一個 3 題體感問卷：
1. 這週入睡比之前快/沒變/變慢？
2. 醒來次數（0 / 1 / 2+ 次）？
3. 早晨疲倦感（清爽 / 普通 / 疲倦）？

依結果建議：3 題都好 → 維持 / 至少 1 題進步 → 維持 + 加 1 加速任務 / 都沒改善 → 退回 Step n-1，加強保健補給。

**這需要 app 端開發**（reflection / pulse-check 模組），不在你內容工作範圍，但內容上可以先標 placeholder 文案。

### 2. 主任務缺「為什麼」說明

加速任務都有「任務目的／體感影響」 column，**主任務沒有**。

主任務是最高 leverage 的習慣，**內在動機需要 why 來支撐**。建議補：

| Type | 主任務「why」建議 |
|---|---|
| 壓力型「建立數位宵禁」 | 藍光抑制褪黑激素、視覺刺激維持高皮質醇；切斷螢幕 = 一次處理兩個生理機轉 |
| 節律型「固定起床時間」 | 起床時間是生理時鐘最強的 zeitgeber（時鐘訊號）；固定起床比固定就寢更有效 |
| 代謝失衡型「固定晚餐結構」 | 蛋白質 + 蔬菜 + 原型澱粉 = 慢消化、穩血糖、減少夜間 cortisol 反彈 |
| 荷爾蒙波動型「夜間降溫」 | 體溫下降是入睡的關鍵訊號；荷爾蒙波動易讓核心體溫不穩 |

### 3. 荷爾蒙波動型主任務考慮擴充

目前主任務「夜間降溫」屬於**物理環境介入**，但 hormonal sleep issues 臨床上**白天活動 + 壓力釋放**證據更強：
- CBT-I（睡眠認知行為治療）
- 規律有氧運動（傍晚或下午）
- 溫度管理（你現在做的）

**建議考慮**：
- 升級「20-30 分鐘輕中度活動」從加速任務 → 平行主任務 of 荷爾蒙型
- 或改寫主任務為「**白天 20-30 分鐘輕中度活動 + 睡前降溫**」雙組合

### 4. 保健品任務多產品的呈現方式

代謝失衡型有 **5 個產品在不同時段吃**（鎂/苦瓜胜肽/肉鹼/B群/甲殼素）。

**app 端設計建議**：
- 1 個「保健補給」task
- 用 **checklist subtasks** 列出時段：[早餐前苦瓜胜肽] [早餐後 B 群+肉鹼] [午餐後極黑生薑黃] [晚餐前甲殼素] [睡前鎂]
- 配 daily reset（每天重置）

對應到我們 Slice F 做的 checklist 架構。**內容你無需改、app 端會這樣處理**。

### 5. 「壓力型主任務 = 數位宵禁」對某些使用者群體可能太極端

睡前 60 分鐘完全不看螢幕對重度上班族 / 父母可能不可行。建議：
- 加一行「若 60 分鐘對你太挑戰，可從 30 分鐘開始」
- 或在 Step 1（睡前 15 分鐘不滑社群）就明確分階段，這已經很好，**可能只是文案要強調「Step 1 就是合格達成」**
