## 0. 一句話現況

正在探索一套**「以『吃』為核心引擎的城市級旅程收集系統」**，靈感主要來自 Pikmin Bloom 的「真實世界 ambient 累積」哲學，但刻意避開花園/植物隱喻（太像 Pikmin、也跟既有花朵模板重複）。

**⭐ 2026-06-01：所有大方向決策已定錨（見 §12）。下一步是拆 slice 寫 spec。** 早期 §5「雙引擎（吃+走）」已收斂為「單引擎（吃）+ 加成層」，走路移除。

## 1. 背景與動機

- 使用者想「大幅升級 UI」，方向是**年輕化、遊戲化，但不要太花**（怕視覺麻痺）、**喜好不一**（畢竟是工具型 app）。
- 重要產品脈絡：**這個習慣模組未來會放進 cofit app**。cofit 強調**飲食紀錄**（拍照）。
  - 飲食紀錄前 8 週較強制，之後不強制但仍鼓勵。
  - 大家到處吃美食、旅行，很多照片本來就是食物 → 天然的真實世界素材。
- 使用者有 **105 萬訂閱的 YouTube 頻道**，系列「初日會客室」邀請健康領域專家。未來這些**專家/醫生/網紅/達人會設計計劃**（免費或付費加入），可為他們設計**專屬徽章**。可能是全球各地的專家。
  - 例：蔡英傑教授 → 腸道改善計劃；文長安教授（食安）→ 降低加工食品添加物計劃。

## 2. 參考對象的取捨（使用者的口味校準）

| App | 使用者評價 | 我們的取用 |
|---|---|---|
| **Pikmin Bloom** | 機制很厲害、很多人玩；「結合現實世界」是它最重要的特色 | **取其哲學**（真實世界 ambient 累積、零懲罰、走過留痕），**不取其視覺**（卡通花花太童趣） |
| **Forest** | 比較均衡，但設計舊了，希望能更突破 | 突破點＝**零懲罰**（Forest 的核心是「分心→樹枯死」的恐懼驅動，已過時）。我們要珍惜驅動、不要恐懼驅動 |
| **Duolingo** | 太強烈了一點 | 不走高刺激、連勝威脅、紅心懲罰那條路 |

校準結論：要站在**「溫和 / 珍惜驅動 / ambient / 工具為主、遊戲在底層」**這一側，遠離「高刺激 / 懲罰驅動 / 遊戲蓋過工具」。

## 3. Pikmin Bloom 機制拆解（供跨 session 參考）

那篇 Yahoo 文章只是操作攻略、沒有心理學分析。真正的設計精髓（我補充的）：

| 機制 | 心理槓桿 |
|---|---|
| 走過開花 | 把抽象努力變成**永久、可回看的美麗痕跡** |
| 零懲罰 | 漏天不失去任何東西。用「珍惜你已擁有的」吸引，不用「你會失去」威脅（vs Duolingo） |
| 被動累積 | 行為發生在真實生活裡，app 只是視覺化，不需額外「玩」 |
| 延遲成長 | 花苗→發芽→成熟，等待製造期待與投入 |
| 陪伴感 | 照顧小生物啟動養育本能、parasocial 依附 |
| 收集多樣 | 1500+ 裝飾，溫和版蒐集多巴胺 |
| 生活敘事 | 每晚回顧路線/照片，運動變成「我的故事」 |

**為什麼讓人持續玩**：正向損失趨避（不想破壞累積的美，但無明確懲罰）＋稟賦效應＋變動獎勵＋零摩擦。

## 4. 收斂過程（決策軌跡）

1. **第一版 mockup（已否決）**：editorial / 文藝風（Noto Serif、cream、身分宣告當 hero）。使用者覺得**不喜歡**，要更年輕、更遊戲化。
2. **花園方向（已否決）**：完成習慣→養花園。使用者覺得**太像 Pikmin**，且與既有花朵模板重複。
3. **提了 5 個非植物方向**：星空/星座、靈光體、同伴化身、圖鑑收藏、旅程地圖。
4. **關鍵洞察（使用者提出）**：**吃和走是普世、終身、人類共通的行為**，適合當**底層引擎**；其他習慣因人而異，是**上層內容**。
5. **雙引擎模型成形**（見下）。
6. **技術約束（使用者提出 + 我確認）**：
   - **不要太強調步數** — 而且 web 瀏覽器本來就拿不到計步器、無法背景追蹤 GPS，技術上也對。步數那種留到 cofit 原生 app。
   - **可以追蹤軌跡，但不要耗地圖 API** — 結論：**用純 SVG 風格化世界，不接任何地圖 API（零成本）**。儲存「軌跡/地點」只是存經緯度數字（免費），成本只在「渲染真實地圖」時才產生，而我們根本不需要真實地圖。城市級插旗即可，不追精確 GPS。
7. **最新 mockup：「旅程世界」**（見 §6）。**待回饋。**

## 5. 雙引擎模型（核心架構）⭐

| 層 | 驅動 | 隱喻角色 | 理由 |
|---|---|---|---|
| **移動引擎** | 🚶 走路 | 帶你探索世界、解鎖新區域 | 走路天生是「移動」 |
| **收集引擎** | 🍽 吃 | 沿途收集回憶（飲食照片＝拍立得/明信片） | 照片是真實世界戰利品 = Pikmin 明信片機制 |
| **個人習慣層** | 睡眠/腸道/壓力…（因人而異）+ 達人計劃 | 「這個旅人正在練什麼功」 | 不必每個習慣都有自己的視覺；它們是當前任務 |

**為什麼這個分工關鍵**：用單一隱喻平等遊戲化所有習慣做不到（習慣太異質）。但只要世界引擎錨在「吃＋走」這兩個普世行為，整套對**每個人一致運轉**；個人習慣再插進來。對要鋪進 cofit 廣大用戶的產品特別重要。

**兩層工作不同**：
- 個人習慣 = 產品的**目的**（為什麼下載）→ 留在乾淨的工具側（每日打卡）
- 吃＋走的世界 = 產品的**留存引擎**（為什麼每天回來）→ ambient、收集、捨不得放
- 這個分工剛好解掉「工具型 app 怕太花」：核心工具乾淨，遊戲化集中在「吃＋走長出的世界」裡。

**達人徽章怎麼接**：達人計劃 = 世界裡的「嚮導路線」。完成蔡英傑教授腸道計劃 → 蓋他的簽名章，收進手帳/護照。底層（吃走）給持久性與容器，上層（達人計劃）給源源不絕的新路線與徽章。

**達人徽章的戰略價值**（成長飛輪）：105 萬訂閱頻道 → 專家設計計劃 → 完成獲簽名徽章 → 截圖炫耀＝免費行銷 → 專家也有動機推廣（品牌曝光）→ 免費/付費＝變現分層 → 全球專家＝護照式收集。徽章把「跟誰學的」變成可見身分標記，呼應既有的身分宣告哲學。

## 7. 仍待決定（回來接續時先處理這 3 題 + 1 視覺）

1. **視覺調性**：目前是「明亮現代插畫」。可選更暖（手作/紙感/拍貼）或更冷（科技感/精品集章）。
2. **走路的角色**：真的影響解鎖（要某種步數/距離輸入），還是純象徵裝飾（完成「散步」打卡就解鎖一格）？傾向後者（不依賴步數）。
3. **個人習慣 vs 世界的關係**：
   - (a) **純底層**：吃＋走 100% 驅動世界，個人習慣只在工具側打卡、不影響世界 → 最乾淨，但個人習慣少獎勵感。
   - (b) **加成模型**：吃＋走是主引擎，完成個人習慣/達人計劃給加成（解鎖特殊地點、稀有徽章、旅人狀態變好）→ 個人習慣也有回饋，但較複雜。
4. **地點精細度**：城市級（拼「吃過的地圖」）vs 更模糊（只分國家/區域）vs 深度 GPS（不建議，web 做不到、且耗資源）。傾向城市級。

## 9. 設計守則（這個方向的不變原則）

1. **工具乾淨、收集分開** — 「今天」分頁純清單；「旅程」分頁才是遊戲化。opt-in、不潑灑、不麻痺。
2. **零懲罰** — 漏天不失去收集，只是沒長新東西。珍惜驅動 ≠ 恐懼驅動。
3. **吃＋走當普世底層引擎** — 不要把所有習慣硬塞進同一個遊戲化視覺。
4. **不接地圖 API、不強調步數** — 純 SVG 風格化世界，城市級插旗。步數/GPS 留給 cofit 原生 app。
5. **結合現實** — 飲食照片＝真實生活回憶，是這個方向的核心、不是裝飾。
6. **達人徽章＝社交貨幣 + 行銷 + 變現飛輪** — 善用 105 萬訂閱頻道。
7. **行為科學 > 遊戲化**（專案原則 #1）— 遊戲化是 ambient 獎勵層，不可蓋過 anchor/identity/嚮往那套行為科學核心。

### 12.2 為什麼座標 = 行為科學（不只是遊戲化）

使用者洞察：**地點 / 場景是習慣養成的關鍵**（context-dependent cue）。座標讓 anchor 從「午餐後」升級成「午餐後 @ 公司」「睡前 @ 家」——context 強化。所以座標是橫切屬性（飲食 log + 一般習慣完成都帶得到），同時餵養世界地圖 + 強化習慣養成。

### 12.3 核心模型（取代 §5 雙引擎）

| 元素 | 內容 |
|---|---|
| **世界地圖** | 你「做過習慣 / 記錄飲食」的城市拼圖（城市級插旗、純 SVG、不接地圖 API）|
| **最豐富的 pin** | 飲食拍立得（cofit 拍照）|
| **其他 pin** | 任何有記座標的習慣完成 → 也插旗 |
| **城市等級** | 該城市累積完成次數 → 村莊 → 城鎮 → 都市（量）|
| **領域地標** | 9 GENESIS+IO 面向各自的主題建築（睡眠=燈塔、腸道=花園、壓力=禪園、運動=體育場…），里程碑升 Lv1→Lv2→Lv3（深度 × 領域）|
| **加成層** | 完成個人習慣 / 達人計劃 → 解鎖特殊地點 ⭐ / 稀有徽章 |
| **達人計劃** | 世界裡的「嚮導路線」，完成蓋親簽 conic-gradient 徽章，收進護照（行銷 + 變現飛輪，善用 105 萬訂閱頻道）|
